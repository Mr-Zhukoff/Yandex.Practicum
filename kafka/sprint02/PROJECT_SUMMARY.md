# Обзор проекта - Kafka Message Filter System

## Выполненные задачи

### ✅ Подзадача 1: Блокировка нежелательных пользователей

**Реализованные компоненты:**
- [`processors/block_manager.go`](processors/block_manager.go) - процессор управления блокировками
- Персистентное хранилище в Goka Group Table: `block-manager-blocked-users-table`
- Топик Kafka: `blocked_users` для команд блокировки/разблокировки
- Автоматическая фильтрация в [`processors/message_filter.go`](processors/message_filter.go)

**Функции:**
- Каждый пользователь имеет свой список заблокированных
- Блокировки хранятся персистентно в Group Table
- Сообщения от заблокированных пользователей автоматически отбрасываются
- Поддержка динамической блокировки/разблокировки

### ✅ Подзадача 2: Цензура на слова

**Реализованные компоненты:**
- [`processors/censor_manager.go`](processors/censor_manager.go) - процессор управления цензурой
- Глобальный список запрещённых слов в Group Table: `censor-manager-censored-words-table`
- Топик Kafka: `censor-actions` для управления словами
- Автоматическая цензура в фильтре сообщений

**Функции:**
- Динамическое добавление/удаление запрещённых слов
- Автоматическая замена запрещённых слов на `***`
- Регистронезависимое сравнение
- Все сообщения проходят через цензуру перед отправкой

## Технологический стек

- **Язык:** Go 1.21+
- **Stream Processing:** Goka (https://github.com/lovoo/goka)
- **Kafka Client:** IBM Sarama
- **Message Broker:** Apache Kafka 7.5.0
- **Coordination:** Apache Zookeeper
- **Containerization:** Docker & Docker Compose
- **Monitoring:** Kafka UI

## Ключевые особенности реализации

### Персистентное хранение (Group Tables)

Используются Goka Group Tables для хранения состояния:
- Список заблокированных пользователей для каждого пользователя
- Глобальный список запрещённых слов
- Автоматическая репликация и восстановление

### Stream Processing

Три независимых процессора:
1. **Block Manager** - обрабатывает команды блокировки
2. **Censor Manager** - обрабатывает список запрещённых слов
3. **Message Filter** - фильтрует сообщения, используя Lookup в таблицы

### Масштабируемость

- Поддержка партиций Kafka
- Возможность горизонтального масштабирования
- Эффективное использование ключей для партиционирования

2. Логи процессора показывают все операции
3. Консьюмер отображает отфильтрованные сообщения

## Тестовые сценарии

Автоматический тестовый продюсер ([`cmd/producer/main.go`](cmd/producer/main.go)) выполняет:

1. ✅ Добавление запрещённых слов (badword, spam, offensive)
2. ✅ Отправка обычных сообщений (проходят без изменений)
3. ✅ Отправка сообщений с запрещёнными словами (цензурируются)
4. ✅ Блокировка пользователя (Bob блокирует Charlie)
5. ✅ Отправка от заблокированного (не проходит)
6. ✅ Отправка от незаблокированного (проходит)
7. ✅ Разблокировка пользователя
8. ✅ Отправка после разблокировки (проходит)
9. ✅ Удаление запрещённого слова
10. ✅ Проверка удалённого слова (не цензурируется)
